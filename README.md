## Implementation Guidelines

### Directory Structure
Each language should follow this structure:
```
[language]/
├── strings/
│   ├── reverse_string.[ext]
│   ├── palindrome.[ext]
│   └── ...
├── arrays/
│   ├── second_largest.[ext]
│   ├── rotate_array.[ext]
│   └── ...
├── stacks/
│   ├── valid_parentheses.[ext]
│   └── ...
├── sets/
│   └── ...
├── maps/
│   └── ...
└── README.md (language-specific documentation)
```

### Code Standards
- Each algorithm should be implemented as a standalone function/method
- Include comprehensive test cases for each implementation
- Follow language-specific naming conventions and best practices
- Add clear documentation and comments explaining the algorithm
- Include time and space complexity analysis

### Testing
- Unit tests should accompany each algorithm implementation
- Test cases should cover edge cases, normal cases, and error conditions
- Use appropriate testing frameworks for each language
